1. Visão geral

O ClaOdin-Quiz é um aplicativo web de perguntas e respostas no estilo Kahoot, feito para usar em sala de aula. O professor cria as perguntas, projeta numa tela, e os alunos respondem pelos próprios aparelhos (celular/notebook) entrando por um PIN — sem instalar nada, apenas abrindo um endereço no navegador.

Principais recursos

  • Modos de jogo: solo, dupla e grupo (até 5 integrantes por equipe).
  • Apelido + nome real: os colegas veem só o apelido/nome da equipe; o professor vê os nomes reais nas estatísticas para lançar a nota.
  • Pontuação por questão: ⚡ rápido (mais rápido = mais pontos) ou 🐢 lento (acertou = pontos cheios).
  • Editor de quizzes salvo no servidor: criar, editar, consultar e lançar a qualquer momento.
  • Editor de texto rico: barra de formatação (negrito, itálico, listas, sub/sobrescrito…), modo de código HTML e inserção de equações LaTeX renderizadas por MathJax (servido localmente, sem internet). novo
  • Até 10 alternativas por pergunta (padrão 4), cada uma com forma e cor próprias (triângulo, losango… estrela, lua, sol, raio). novo
  • Importar quizzes no formato GIFT (Moodle): texto, alternativas e a correta — convivendo com equações LaTeX. novo
  • Imagens nas perguntas (enviar arquivo, colar print ou URL).
  • Geração por IA: cria perguntas sobre um tema usando Claude, ChatGPT ou Gemini (à escolha), com seletor de quantas alternativas por pergunta.
  • Ranking animado, contadores ao vivo, controle total do professor e reconexão automática (se o celular dormir, o aluno volta no mesmo ponto). Na revelação, o professor vê quem está online e quem caiu, para esperar antes de avançar. novo
  • Logo da ETE Epitácio Pessoa no login, na espera do aluno e nas telas do professor (revelação e resultado final).

2. Onde está salvo & como rodar

Pasta do projeto

C:\Users\erly_\projetos\quiz-app

Comandos (rodar no terminal, dentro dessa pasta)

npm install        # 1ª vez: instala as dependências
npm run build      # compila o app (checa tipos + gera client/dist)
npm start          # liga o servidor (serve tudo na porta 3000)

# para desenvolver com recarga automática:
npm run dev
⚠️ Mantenha o servidor de pé durante a aula. Deixe a janela do terminal com npm start aberta. Enquanto ela estiver aberta, o servidor não cai. As salas ficam na memória — se o servidor reiniciar, os jogadores precisam reentrar.

Endereços de acesso

  • No PC do professor: http://localhost:3000
  • Celulares/alunos (mesma rede Wi-Fi): http://192.168.0.2:3000 o IP pode mudar
ℹ️ O IP da rede pode mudar (é distribuído pelo roteador). O servidor imprime o IP atual no terminal toda vez que liga. Veja a seção Operação para deixá-lo fixo.

6. Funcionalidades em detalhe

🎯 O loop do jogo

O professor abre uma sala (recebe um PIN e mostra na tela). Os alunos entram com o PIN. O professor escolhe o modo e clica em iniciar. Para cada pergunta:

  1. A pergunta aparece em todos com um relógio regressivo; o professor vê quantos estão online e quantos já responderam.
  2. Quando o tempo acaba, todos respondem, ou o professor clica "Pular", aparece a resposta correta + um botão "Continuar".
  3. Em seguida vem a tela do ranking animado (os nicks sobem/descem), com a lista de presença (online/desconectado) + botão "Próxima pergunta".
  4. No fim: pódio + tabela de estatísticas com os nomes reais (precisão, acertos, tempo médio) para o professor lançar a nota.

👥 Modos solo / dupla / grupo

Modelo "1 aparelho por equipe": no login, a equipe digita o nome da equipe + os nomes reais dos integrantes (solo = 1; dupla = 2; grupo = 2 a 5). Todos da equipe recebem a pontuação da equipe.

Importante: escolha o modo antes de os alunos entrarem (o formulário de entrada se adapta ao modo).

🏆 Pontuação por questão

  • ⚡ Rápido — acerto + rapidez: 1000 pontos (instantâneo) caindo até 500 (no limite do tempo).
  • 🐢 Lento — acertou vale a pontuação cheia (1000), sem importar o tempo.

É escolhido em cada questão no editor.

🔌 Reconexão (resolve "celular dormindo")

Cada aluno tem uma identidade estável. Se a conexão cai (tela bloqueia, Wi-Fi oscila), ele não é removido da sala — aparece o aviso "reconectando" e, ao voltar, ele retorna na mesma pergunta, com o mesmo nome, sem precisar logar de novo. Vale também no lobby (antes de iniciar).

🔒 Segurança

  • Senha de professor novo: toda a área do professor (quizzes, gabaritos, estatísticas, geração por IA, import e PDF) exige login. O token de sessão usa um segredo do servidor (não dá para forjar) e o login tem proteção contra força bruta. Configure a senha em PROFESSOR_PASSWORD no .env.
  • Só o dono da sala controla o jogo (alunos não conseguem iniciar/avançar).
  • As estatísticas com nome real são protegidas por um token — só o professor da sessão acessa.
  • Validação das entradas; um erro inesperado não derruba o servidor.
  • Anti-XSS no conteúdo rico: como enunciados/alternativas aceitam HTML e são exibidos no aparelho de outros alunos, todo HTML é sanitizado no servidor (sanitize-html) e no navegador (DOMPurify) antes de aparecer. novo
  • Limite de equações para evitar travar o navegador da turma, e a importação GIFT é resistente a arquivos malformados/gigantes. novo

🖼️ Imagens nas perguntas

No editor, cada pergunta pode ter imagem por enviar arquivo, colar (Ctrl+V de um print) ou URL. O arquivo vai para o servidor (até 4 MB) e aparece na tela do aluno e no projetor.

✨ Geração por IA

No editor há o painel "Gerar com IA": você escolhe o modelo (Claude / ChatGPT / Gemini), o tema, a quantidade, o nível e quantas alternativas cada pergunta terá, e a IA cria as perguntas (você revisa antes de salvar). Veja a configuração na seção IA & chaves.

📝 Editor de texto rico (formatação, HTML e LaTeX)

O enunciado tem um editor com barra de ferramentas (negrito, itálico, sublinhado, tachado, sobrescrito/subscrito, títulos, listas, link). Há um botão </> HTML para editar o código HTML direto, e botões ∑ Equação / ∑ bloco para inserir LaTeX. Uma pré-visualização ao vivo mostra o resultado renderizado. As alternativas também aceitam HTML e LaTeX.

Matemática sem internet: o LaTeX é renderizado pelo MathJax servido localmente (rota /mathjax) — funciona na rede da escola sem depender de CDN. Use $ ... $ para equação na linha e $$ ... $$ para equação em destaque.

Seguro por padrão: todo HTML do usuário passa por sanitização (DOMPurify no navegador e sanitize-html no servidor) antes de ser exibido, removendo scripts maliciosos — mas preservando as equações.

🔢 Alternativas: de 2 a 10 por pergunta

Cada pergunta começa com 4 alternativas e o botão + Adicionar alternativa permite chegar a 10 (e remover de volta até 2). Cada alternativa tem uma forma e cor próprias (triângulo, losango, círculo, quadrado, estrela, lua, sol, coração, hexágono, raio) para reconhecimento rápido e acessível.

📥 Importar do GIFT (Moodle)

No editor, o botão 📥 Importar GIFT lê um arquivo .gift e carrega as perguntas (texto, alternativas e a correta) para você revisar antes de salvar. Suporta múltipla escolha, Verdadeiro/Falso, títulos, comentários e os escapes do formato.

LaTeX × GIFT sem conflito: o GIFT usa = para marcar a resposta correta — o mesmo sinal que aparece em equações como $x = 2$. O importador isola os blocos de matemática antes de interpretar a estrutura do GIFT e os restaura no fim, então o = (e também { }) dentro das equações nunca é confundido com a sintaxe do GIFT.

👀 Presença na revelação (esperar quem caiu)

Quando a pergunta é respondida, a tela do professor mostra os apelidos dos alunos online e dos desconectados (estes marcados com ⟳ em destaque), além de um aviso. Assim o professor só passa para a próxima pergunta com a certeza de que quem caiu teve tempo de voltar.

🏫 Logo da escola

A logo da ETE Epitácio Pessoa aparece no login, na tela de espera do aluno e nas telas do professor (resposta revelada e resultado final). É uma versão vetorial nas cores do tema; para usar o arquivo oficial, basta salvá-lo em client/public/logo-escola.png (instrução comentada em SchoolLogo.tsx).

8. Tempo real (Socket.IO)

O jogo ao vivo usa WebSocket. Resumo dos eventos:

Do cliente para o servidor

  • Professor: host:create, host:setMode, host:start, host:reveal, host:next, host:rejoin.
  • Aluno: player:peek (descobrir o modo), player:join, player:answer, player:rejoin.

Do servidor para o cliente

  • question:show, answer:progress, question:reveal, player:result, lobby:update, game:ended, presence.

9. IA & chaves de API

A geração de perguntas usa três provedores possíveis. Cada um é opcional — o editor mostra só os que têm chave. O resto do app funciona sem nenhuma IA.

Onde ficam as chaves

Num arquivo .env na raiz do projeto (não vai para o git — está no .gitignore). Há um modelo em .env.example. Estrutura:

# Claude (Anthropic) — console.anthropic.com
ANTHROPIC_API_KEY=...

# ChatGPT (OpenAI) — platform.openai.com
OPENAI_API_KEY=...

# Gemini (Google) — aistudio.google.com/apikey  (normalmente começa com "AIza...")
GEMINI_API_KEY=...

# Opcional: trocar o modelo de cada provedor
# ANTHROPIC_MODEL=claude-haiku-4-5   (mais barato)
OPENAI_MODEL=gpt-5.4-mini            # configurado atualmente
GEMINI_MODEL=gemini-2.5-flash        # configurado atualmente

Após editar o .env, reinicie o servidor. corrigido O modelo de cada provedor agora é lido a cada requisição — antes, por um problema de ordem de carregamento do .env, a troca de modelo não tinha efeito.

10. Operação (rede)

Servidor sempre de pé

Para a aula não cair, rode npm start numa janela de terminal e deixe aberta. O servidor mostra o IP atual ao ligar.

Firewall

Foi criada uma regra no Firewall do Windows liberando a porta 3000 para entrada (sem isso, os celulares não conseguem conectar, mesmo na mesma rede).

IP fixo (recomendado)

O IP é distribuído por DHCP e pode mudar (já foi 192.168.0.2). Para o link dos alunos não quebrar:

  • Reserva de DHCP no roteador (associe o PC a um IP fixo) — jeito mais limpo; ou
  • IP estático direto no Windows (Configurações → Rede → Ethernet).

Mesma rede

O celular do aluno precisa estar no mesmo Wi-Fi/roteador do PC. Alguns roteadores têm "isolamento de clientes" (AP isolation) que impede um aparelho de falar com outro — se não conectar, verifique essa opção no roteador.

11. Como usar (passo a passo)

Professor

  1. Abra http://localhost:3000, clique em Sou professor e digite a senha do professor (padrão professor — troque em PROFESSOR_PASSWORD no .env). Você fica logado nesse aparelho.
  2. Em 📚 Meus quizzes: crie um Novo quiz (ou edite um existente). No editor você pode digitar as perguntas, enviar/colar imagens, marcar a alternativa correta, definir tempo e pontuação, e usar ✨ Gerar com IA.
  3. Salve e clique em ▶ Iniciar jogo.
  4. Escolha o modo (Solo/Dupla/Grupo) e mostre o PIN aos alunos.
  5. Quando todos entrarem, clique em Iniciar quiz. Conduza com "Revelar / Continuar / Próxima".
  6. No fim, veja a tabela com os nomes reais e as notas.

Aluno

  1. Escaneie o QR code mostrado na tela do professor → o celular abre já na entrada, com o PIN preenchido. (Sem QR: abra http://IP-DO-PROFESSOR:3000, toque em Sou aluno e digite o PIN.)
  2. Confirme/preencha o apelido (ou nome da equipe) + os nomes reais.
  3. Responda às perguntas tocando na alternativa.

# Constroi a imagem e sobe em segundo plano:
docker compose up -d --build
⏳ A primeira vez demora (baixa o Node e o LaTeX e compila o front — alguns minutos e várias centenas de MB). Da segunda vez em diante é quase instantâneo.

3) Acesse e opere:

  • No próprio PC do professor: http://localhost:3000
  • Nos celulares (mesma rede): http://IP-DO-PC:3000 — descubra o IP com ipconfig (Windows) ou ip addr (Linux). No Docker, ignore o IP que o log do servidor imprime (é o IP interno do contêiner); use o IP do PC.
  • Ver o log: docker compose logs -f · Parar: docker compose down · Subir de novo: docker compose up -d
💾 Os dados ficam salvos em server/data/ no seu PC (banco, imagens enviadas e estatísticas das turmas), via volume do Docker — não se perdem ao parar/atualizar o contêiner.
🔑 TROQUE a senha do professor. Toda a área do professor (quizzes, gabaritos, estatísticas, PDF com nomes reais) é protegida por uma senha. O padrão é professor — defina a sua em PROFESSOR_PASSWORD no .env, senão os alunos podem entrar e ver os gabaritos. Vale para qualquer modo (manual, Docker, rede).
📱 QR code no Docker: o QR de entrada usa o IP/porta do servidor. Dentro do contêiner o IP detectado não serve para a rede, então defina PUBLIC_HOST no .env com o IP do PC (ex.: 192.168.0.2) para o QR apontar certo. (Na instalação manual, o IP é detectado automaticamente.)

14.4 — Rodar sem Docker (instalação manual)

  1. Instale o Node.js 20+ (nodejs.org).
  2. (Opcional, só p/ o PDF) Instale o LaTeX: MiKTeX no Windows (miktex.org) ou, no Linux: 
    sudo apt install texlive-latex-recommended texlive-latex-extra texlive-lang-portuguese
  3. Na pasta do projeto: 
    npm install
npm run build
npm start
  4. Abra http://localhost:3000. (Para desenvolver com recarga automática: npm run dev.)

14.5 — Acesso na rede da escola

  • Firewall: libere a porta 3000 para entrada (no Windows já existe a regra; em outro PC, crie de novo).
  • Mesma rede Wi-Fi entre o PC e os celulares; cuidado com "isolamento de clientes" (AP isolation) no roteador.
  • IP fixo (reserva de DHCP no roteador) evita o link dos alunos quebrar — ver a seção Operação.

14.6 — Atualizar e fazer backup

  • Atualizar (após um git pull com novidades): docker compose up -d --build reconstrói e troca o contêiner sem perder os dados.
  • Backup: copie a pasta server/data/ (contém o banco quiz.sqlite, os uploads e os arquivos de estatística das turmas).
